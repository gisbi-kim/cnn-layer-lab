# CNN 웹 실습 진행 노트

## 목표

중학생이 CNN을 수식보다 관찰 중심으로 이해하도록 한다.

- 이미지는 28x28 숫자 배열이다.
- CNN은 작은 필터로 선, 획, 부분 모양을 찾는다.
- 레이어를 지날수록 원본 픽셀보다 추상적인 특징으로 바뀐다.
- 같은 구조라도 MNIST와 Fashion-MNIST는 난이도가 다르다.

## 권장 흐름

| 시간 | 활동 | 교사 발문 |
| --- | --- | --- |
| 0-10분 | MNIST 이미지와 라벨 보기 | 사람은 어떤 단서를 보고 숫자를 알아보는가? |
| 10-20분 | CNN 구조 설명 | Conv, Pool, Dense가 각각 무슨 일을 하는가? |
| 20-35분 | 브라우저에서 학습 실행 | Loss가 내려간다는 것은 무엇을 뜻하는가? |
| 35-50분 | 레이어별 feature map 관찰 | 첫 레이어와 두 번째 레이어의 그림은 어떻게 다른가? |
| 50-70분 | 학습 샘플 수, epoch 바꿔보기 | 데이터가 적으면 왜 결과가 흔들리는가? |
| 70-90분 | Fashion-MNIST 과제 안내 | 숫자보다 옷이 어려운 이유를 evidence로 설명해보자. |

## 학생 실습 질문

1. 학습 전 예측 확률은 왜 거의 무작위처럼 보일까?
2. 학습 후 가장 높은 확률의 숫자는 정답과 일치하는가?
3. `conv1_edges`에서는 어떤 선이나 획이 보이는가?
4. `conv2_parts`에서는 원본 숫자와 다른 어떤 조각들이 보이는가?
5. 학습 샘플 수를 512로 줄이면 accuracy가 어떻게 변하는가?
6. epoch를 늘리면 항상 좋아지는가?

## Fashion-MNIST 과제

학생들에게 `index.html`을 복사해 `fashion.html`을 만들게 한 뒤 다음을 바꾸게 한다.

```js
const LABELS = [
  'T-shirt/top', 'Trouser', 'Pullover', 'Dress', 'Coat',
  'Sandal', 'Shirt', 'Sneaker', 'Bag', 'Ankle boot'
];
```

Fashion-MNIST는 MNIST와 같은 28x28 흑백 이미지 구조를 사용하므로 CNN 구조를 거의 그대로 쓸 수 있다. 다만 브라우저용 데이터 로더는 현재 MNIST 스프라이트 파일에 맞춰져 있으므로, 과제로는 다음 중 하나를 선택한다.

- 교사가 Fashion-MNIST를 같은 스프라이트/one-hot 라벨 형식으로 변환해 제공한다.
- 학생은 Colab/Python에서 Fashion-MNIST를 학습하고, 웹 실습 결과와 비교 보고서를 작성한다.
- 심화 학생은 `fetch`로 gzip IDX 파일을 읽는 로더를 직접 구현한다.
